# 静态资源文件

静态资源文件就是static目录下的文件。静态资源文件是通过link或script引入的，也包括css中通过url引入的静态资源。

静态资源没有太多要说的内容，有一个需要注意的是static/compoents目录，这个目录是每个组件使用的静态资源，这个设计来自于具体的需求：

- 开发者会把需要模块化引入的静态文件在组件目录一并封装发布。

- 前端的应用打包以及服务端处理希望可以统一使用一个路由提供静态资源，通常是/static/*路由。

- 前端应用打包希望直接把打包后的文件和静态资源目录一并交给服务端并且只提交转码压缩处理后的代码。

当进行应用构建时，会自动把组件的静态资源复制到static/components目录下，比如：

```
static/components/x-user/user.js
```
